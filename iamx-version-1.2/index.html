SpartaMaps-1

Project Info

Year - February 2016 - May 2016

Link  -  SpartaMaps

GitHub  -  SpartaMaps Git

Project Report  -  SpartaMaps Report


Description

SpartaMaps is a extensive project about the San Jose State University campus, which is designed to offer interactive navigation and much more interesting features to varied users of the application. The project overview is organized into part 1 and part 2 to discuss the features in detail.

SpartaMaps-1 showcases some of the key features of SpartaMaps such as

  • Bootstrap Homepage
  • Interactive Dashboard
  • Health Statistics
  • Friends Finder
  • Social Network Integration
  • Google Analytics


Wireframes


Highlights

  • The Homepage is developed using the bootstrap component, making the site responsive. A login feature is enabled using sessions and there is also an option for Single Sign On using social network integration.
  • The Dashboard is interactive with various eye-pleasing graphical components such as charts rendered using d3.js. The events are being streamed from the university website in real-time. The profile of an user can be edited when needed. There are options for uploading pictures, accessing the calender, adding events, showing various stats such as most visited places, most favorite friends and social media integration.
  • Health Statistics is displayed for each person, showing the distance travelled, the steps walked and the calories burnt using simple mathematical formula
  • Friends Finder is a feature developed to connect with different persons around the campus. This feature was developed using Pub-Nub API and MapBox API. The location of different persons are published on the app and any user can connect with any person of their choice using this feature.
  • This project includes integration with many social networks such as Facebook, Youtube, Twitter, Instagram, FourSquare, Yelp etc
  • This project is integrated with Google Analytics API, which enables the admin to know about the application's popularity

Technologies Used  -  HTML/CSS, JavaScript, Three.js, WebGL

GitHub  -  Body Browser

Project Report  -  Body Browser

Link  -  Body Browser

 

Description

The project is a 3d modelling of the living body. Human Body is implemented in detail. In Addition, the project also covers the models of Mammoth, Raven & Lioncub

The project showcases features such as

  • Skeletal System of Human
  • Skeleton System of Mammoth
  • 3D Modelling of Body
  • Models of Male and Female


3D Modelling



Highlights

  • Different models are rendered using Three.Js. A scene is created for the placement of the elements.
  • Display with Three.js requires initialization of the scene, camera and the renderer to render the scene with the camera. The Perspective Camera is used in this project.
  • Other Attributes to be considered are aspect ratio, near and far clipping plane to render the zoom in and zoom out features.
  • The renderer is a WebGL Renderer and the size of the browser window is set for efficient rendering.
  • Then the renderer element is added to the HTML document, which is basically a <canvas> element.

Real-time Twitter Visualization

Project Info

Year - May 2016

Technologies Used  -  Apache Spark, Python, Zookeeper, Apache Kafka, HTML/CSS, d3.js, Twitter API

GitHub  -  Live Streaming Data

Project Report  -  Live Streaming Data

 

Description

The project is a Real-time visualization of Twitter Streaming data using Apache Spark and Kafka.

The project showcases features such as

  • Apache Spark Configuration
  • Visualization using D3.js
  • Real-time Streaming of Twitter Feeds
  • Topic Based Kafka Integration
  • Republicans vs Democrats
  • Centralized Zookeeper Monitoring



Highlights

  • Live Feeds for Twitter are got from Twitter API. The project visualizes the comparison of tweets between the Republicans and Democrats on a 5 second window.
  • The Apache zookeeper is started to monitor the server configuration
  • Kafka server is also started with a unique topic. Kafka consumer is also started. All at the same time.
  • Using Python script, the tweets are sent to the Kafka Broker. The broker receives the tweets, processes the tweets and saves the information in JSON format.
  • D3.js is used to visualize the contents of the JSON file
